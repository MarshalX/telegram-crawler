Top peer rating

If enabled, the rating of top peers indicates the relevance of a frequently used peer in a certain category (frequently messaged users, frequently used bots, inline bots, frequently visited channels and so on).

Schema:

The rate delta is computed by taking the time delta between the last time the user used a certain peer and the last time the rating for that peer was received through contacts.getTopPeers and dividing it by the exponential decay from config.

Specifically, clients should:

  1. On startup, invoke contacts.getTopPeers for all the required peer rate categories, fetching and storing the ratings for all top peers in a local database.
  2. Every time a user opens (for example) chat 123456789 the following operation must be done on the cached top peer info.
    Â 
    dateOpened indicates when was the peer used.
    normalizeRate is an arbitrary time in the recent past (when ratings are received from the server using contacts.getTopPeers and the schema described above, it is the time when they were received).
    config.rating_e_decay is the exponential decay from config. 
  topPeer.rating += e^((dateOpened - normalizeRate) / config.rating_e_decay)
  1. Every time the client needs to display a list of commonly used bots, users, mini apps, call destinations, etc (for example for the global search tab), sort the peers by the rating for the specified category.

Use contacts.toggleTopPeers to enable or disable top peer ratings.
Use contacts.resetTopPeerRating to reset the top peer rating of a certain peer, in a certain category.
