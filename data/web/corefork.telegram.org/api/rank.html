Members in group chats (both basic groups and supergroups) can now add a tag next to their name — to show their role at work, what they study in school, and more.

Use messages.editChatParticipantRank to edit the tag, passing the following params:

  • peer: The group
  • user_id: The user whose rank should be changed. Admins with the manage_ranks admin right » can modify the rank of any member.
    Everyone else can only modify their own rank, but only if either:
    • The chat's default rights allow using edit_rank AND/OR
    • The user's banned rights allow using edit_rank
  • rank: The new tag (max 16 chars, no emojis)

The tag will be available in the following places for supergroups:

  • In the message.from_rank flag of all messages sent by group members
  • In the ChannelParticipant.rank flag, fetchable using channels.getParticipant/channels.getParticipants

The tag will be available in the following places for basic groups:

  • In the ChatParticipant.rank flag, available in chatFull.participants, must always be locally cached »
  • Changes to the tag of users will be emitted as updateChatParticipantRank updates (only for basic groups, supergroups don't need this as message.from_rank will always contain the updated rank)
  • message.from_rank will not be populated for basic groups: clients should instead extract tags from ChatParticipant.rank for all basic group members

Admins also have a tag, which is always colored with the peer's color », to distinguish them from non-admins.

Only for admins and the creator, clients should default to showing "Owner"/"Admin" for channelParticipantCreator/channelParticipantAdmin if the rank flag is not set.

Both messages.editChatParticipantRank and channels.editAdmin can be used to edit the tags of supergroup admins.
